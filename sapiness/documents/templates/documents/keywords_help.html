About the dump

This is the file Abippity is currently using to parse individual commands.

Adding support for a new ABAP command starts with adding an entry to this file.

It's largely derived from help.sap.com's ABAP - Keyword Documentation and modified to be more directly parsable.

It's some kind of pseudo Backus-Naur grammar:

  • [ ... ] is optional
  • A | B denotes a choice
  • { ... } is just for grouping
  • Keywords are denoted by UPPERCASE (though ABAP and Abippity are case-insensitive)
  • Non-alphanumeric tokens are also keywords, including ( and )
  • Capturable variables are denoted by lowercase
  • References to sub-grammars are also denoted by lowercase
  • Boolean captures look like: +var
  • List captures look like: var+
  • Renamed variable captures look like: newname:originalvar
  • <at> is a special symbol matching things like: / /pos /pos(len)
